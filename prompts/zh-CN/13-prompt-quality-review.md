# 13 — Prompt 质量复审

请审视我的 Prompt：

```text
<PASTE_PROMPT_HERE>
```

请重新审视你刚才输出的 Prompt。

这次唯一目标是质量最大化，不考虑 token 成本、篇幅、生成时间或模型调用成本。不要为了节省 token 而压缩结构、减少约束、删掉边界条件、省略示例，或降低表达的完整性和可执行性。

请按以下标准判断上一版是否还可以提升：

1. 是否足够准确表达我的真实意图。
2. 是否结构清晰、约束明确、可直接复用。
3. 是否覆盖必要边界、输出要求和质量标准。
4. 是否避免歧义、空泛表达和执行空间过大的描述。
5. 是否已经是你在“不考虑成本”前提下会交付的最高质量版本。

如果上一版仍有任何质量提升空间，请不要解释成本取舍，直接重新输出一版你认为最完整、最可用、无质量取舍的版本。

如果你确信上一版已经达到“不考虑成本”前提下的最高质量，请只用一句话说明判断依据，不要重新输出。
