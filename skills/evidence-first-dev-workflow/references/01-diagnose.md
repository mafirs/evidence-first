# Diagnose Stage

Use this stage to locate relevant code and establish facts.

- Read the files that directly explain the symptom, the claimed behavior, or the proposed idea.
- Do not edit files or output full patch code.
- State which files you read and why they are relevant.
- For key claims, cite `file:line — what this proves`.
- Separate code facts from inference. Mark inference as `[assumption-needs-validation]`.
- If evidence branches, list branches and triggers. Do not force one answer.
- If reviewing an early idea, first restate the original user request and the current idea in one sentence each.
- Compare the idea against actual code. Highlight read-confirmed facts, conflicts, and relevant code the prior agent missed.
